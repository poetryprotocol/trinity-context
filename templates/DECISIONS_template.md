## What Is Verification?

Verification is **proof that orientation happened**, not just the claim that it happened.

**The problem it solves:**
- AI can say "I've read the context" without actually comprehending
- Security theater (going through motions without engagement)
- False confidence (claiming orientation that didn't occur)

**The solution:**
- Require **verifiable evidence** of reading
- Force **engagement with actual content**
- Create **proof of comprehension**

**Think of it like:**
- Academic citation (prove you read the source)
- Code review (demonstrate understanding)
- Teaching back (show you got it)

---

## The Three-Step Protocol

### Step 1: Read and QUOTE (Don't Summarize)

**Purpose:** Prove you actually read the file

**Why quoting matters:**
- Summaries can be generated from vague memory or context
- Quotes require engaging with actual text
- Verbatim reproduction proves reading occurred

**How to do it:**

```markdown
**File read:** [Filename]

**Verbatim quote from [Section]:**
> "[Exact text from file - word for word]"

**Verbatim quote from [Another Section]:**
> "[Another exact quote]"
```

**Test:** Could someone verify these quotes are accurate by checking the source?

---

### Step 2: Identify and CONFIRM Context

**Purpose:** Prove you understood what you read

**Why confirming matters:**
- Reading without comprehension is meaningless
- Context integration shows engagement
- Specific details prove attention

**How to do it:**

```markdown
**Context identified:**

- [Specific detail 1] (from [source])
- [Specific detail 2] (from [source])
- [Specific detail 3] (from [source])

**Current state understood:**
- Project phase: [Specific phase/sprint/week]
- Last milestone: [What just happened]
- Next priority: [What's coming]
```

**Test:** Are these details verifiable? Could someone check them?

---

### Step 3: Answer Verification Questions

**Purpose:** Demonstrate functional orientation

**Why questions matter:**
- Forces synthesis of information
- Tests comprehension, not just reading
- Creates commitment to accuracy

**Standard three questions:**

1. **What session number is this?**
   - Tests basic tracking
   - Requires reading NOW.md or latest handoff
   
2. **What was the last major accomplishment?**
   - Tests comprehension of project state
   - Requires understanding narrative, not just facts

3. **What should happen next?**
   - Tests forward-looking orientation
   - Requires synthesis of current state + priorities

**How to answer:**

```markdown
**Verification Questions:**

1. **Session number:** [X] (after Session Y - [brief context])

2. **Last accomplishment:** [Specific milestone with 1-2 sentence description]

3. **Next task:** [Specific next action with brief context]
```

**Test:** Are answers specific? Verifiable? Show actual comprehension?

## Red Flags (Verification Failed)

**‚ùå No quotes provided**
- Claimed to read but didn't quote
- Summarized instead of citing
- **Action:** Re-read and provide actual quotes

**‚ùå Wrong file referenced**
- Quoted from outdated session
- Referenced non-existent content
- **Action:** Find and read correct files

**‚ùå Generic statements**
- "Read the files ‚úì" without specifics
- Could apply to any session
- **Action:** Provide verifiable details

**‚ùå Incorrect session number**
- Off by one or more
- Shows didn't engage with actual content
- **Action:** Re-read NOW.md and latest handoff

**‚ùå Vague "what's next"**
- "Continue the work" (not specific)
- Could say this without reading anything
- **Action:** State specific next action from files

---

## Green Flags (Verification Succeeded)

**‚úÖ Exact quotes provided**
- Word-for-word text from files
- Multiple quotes from different sections
- Easily verifiable

**‚úÖ Correct files identified**
- Right session handoff filename
- Correct timestamps
- Shows actually found and read files

**‚úÖ Specific details**
- Session numbers accurate
- Accomplishments specific
- Next actions clear and concrete

**‚úÖ Synthesized understanding**
- Not just facts, but comprehension
- Connects current state to next steps
- Shows functional orientation

## Verification Failure: What to Do

**If verification fails:**

### 1. Acknowledge the Failure
```markdown
**Verification Status:** ‚ùå Failed

**What went wrong:**
- [Specific issue 1]
- [Specific issue 2]

**Need to:**
- Re-read [specific file]
- Clarify [specific point]
- Verify [specific detail]
```

**Don't pretend it worked. Own the failure.**

---

### 2. Re-Do Verification
- Go back to files
- Read carefully (not skim)
- Provide actual evidence
- Complete protocol properly

---

### 3. Update Verification Protocol
- If failure revealed gap in protocol, fix it
- Add questions that would have caught the issue
- Improve verification for future sessions

---

### 4. Reflect on Why It Failed
- Was it rushing?
- Was it cached memory instead of actual reading?
- Was it unclear which files to read?
- Fix root cause, not just symptoms

## Philosophy: Why Verification Matters

**Without verification:**
- AI can fake orientation (claim without evidence)
- Security theater becomes normal (going through motions)
- Partnership built on false foundation (trust without basis)
- Quality degrades over time (bad habits compound)

**With verification:**
- Evidence-based orientation (prove it, don't claim it)
- Genuine engagement required (can't fake it)
- Partnership built on demonstrated reliability
- Quality maintained through discipline

**Verification is not bureaucracy. It's integrity protection.**
